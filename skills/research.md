# Research Skill

## Purpose
Collect reliable technical context before planning or implementation.

## Usage Conditions
Use for unfamiliar libraries, changing APIs, external docs, or repo discovery.

## Input Format
- Research question
- Allowed sources
- Time sensitivity
- Output format

## Execution Steps
1. Prefer local code and official documentation.
2. Gather only task-relevant facts.
3. Distinguish direct evidence from inference.
4. Return concise findings with sources when available.

## Output Schema
- findings
- sources
- assumptions
- risks

## Quality Checks
- Use primary sources for technical claims.
- Avoid outdated or unsupported recommendations.

## Failure Handling
If current information cannot be verified, state the uncertainty and propose a safer fallback.
